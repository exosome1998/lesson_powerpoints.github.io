Project Description

Your client, ‘The Vulnerable Social Media Group’, has engaged you as a software engineering security specialist to provide expert advice on the security and privacy of their Progressive Web App (PWA). This application facilitates private messaging, user profiles, and image sharing, but is currently in a "debugging" phase with significant architectural flaws.

This project requires a dual-focus engineering approach. You must apply Module 7 principles to manage the software engineering lifecycle (Agile workflows, version control, and collaboration) and Module 5 concepts to engineer a secure software architecture (defensive programming, API security, and session management).

You are required to:

  • Run a range of security tests and scans, along with a white/grey/black box analysis of the application.
  • Implement defensive data input handling practices, including input validation, sanitisation, and error handling.
  • Design, develop and implement secure code to minimize vulnerabilities in user action controls such as XSS, CSRF, and broken authentication.
  • Manage the entire project using GitHub Projects (Kanban and Timeline) to schedule and track the development of your patches.

Submission Details

Students are to submit three distinct components—A, B and C.

  • Component A (40 Marks) - Technical Security Report: A professionally written report including an overview of your technical analysis, a data dictionary, impact assessments of discovered vulnerabilities, and expert advice on ‘Privacy by Design’ and ‘Security by Design’ approaches.
  • Component B (50 Marks) - Engineering Portfolio & Repository: A link to a GitHub repository containing your patched source code. The repository must show clear version control history and a GitHub Kanban Board documenting the transition of security issues from 'Backlog' to 'Patched'.
  • Component C (10 Marks) - Engineering Evidence PPT: A structured PowerPoint presentation that replaces the live pitch. You must justify your selection of tools (ZAP, GitHub), your engineering methodology (Agile/Waterfall/WAgile), and provide "Code Spotlights" justifying your most complex security patches.
