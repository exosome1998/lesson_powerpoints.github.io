Part A: Operational Scenarios (Multiple Choice)

1. CI/CD Pipeline Integration
Scenario: A fintech startup is building a payment gateway. The DevSecOps lead wants to integrate security testing into their GitHub Actions pipeline so that vulnerabilities are detected in the source code before the program is even compiled or deployed.
2. Healthcare Data Architecture
Scenario: A hospital is developing a patient portal. The lead architect insists that data masking, encryption-at-rest, and minimal data collection are built into the initial database schemas from day one, rather than patching them in after the app is built.
3. Malware Analysis Automation
Scenario: An enterprise email server receives thousands of attachments daily. To prevent ransomware, the architecture automatically executes and analyzes suspicious executable files in a tightly isolated, restricted virtual environment before delivering the email to the user.
4. E-Commerce Vulnerability
Scenario: A malicious user posts a product review on an e-commerce site. Instead of text, they input hidden HTML and JavaScript tags. When other customers view the product page, the script executes in their browser and steals their active login cookies.
5. Cloud Infrastructure Outage
Scenario: A massive power grid failure takes down a cloud provider's primary data center in Sydney. Within seconds, the system's architecture automatically fails over to a secondary data center in Melbourne, ensuring the bank's services experience zero downtime.

Part B: Architectural Analysis (Short Response)

6. User Capability & Secure Design
Scenario: You are designing a mobile banking app for a demographic of elderly users who often use weak, easily guessed passwords (like "password123") and struggle with complex password rotation rules.

Describe how this specific end-user capability influences your secure design features (e.g., what alternative security measures would you implement?).

7. Defensive Error Handling
Scenario: A logistics company's tracking API crashes when a user types letters instead of numbers. The web page currently displays the raw Python stack trace and internal database table names to the user.

Explain why this is a severe architectural security risk and how defensive error handling should be implemented to fix it.

8. Broken Session Management
Scenario: An online streaming platform notices that if a user logs in on a public library computer, closes the browser window (without clicking 'Logout'), and walks away, the next person to open the browser is automatically logged into the first user's account.

Describe how the system's session management architecture must be changed to prevent this vulnerability.

9. Enterprise ROI on Security
Scenario: A government defense contractor is hesitant to invest heavily in a "security by design" approach, arguing that it will slow down initial software development and cost too much upfront.

Evaluate two long-term benefits to the enterprise of implementing these safe and secure development practices that justify the initial cost.
