Part C: Secure Architecture (Short Response)

11. EVALUATE Role-Based Access Control [5 Marks]
Scenario: A hospital is upgrading its software architecture. Previously, all staff shared a generic login. They are planning to implement Role-Based Access Control (RBAC) where doctors, nurses, and administrative staff have distinct permission levels.

Evaluate the implementation of RBAC in this healthcare scenario in terms of system security and operational efficiency.

12. JUSTIFY Parameterized Queries [4 Marks]
Scenario: An e-commerce backend previously used string concatenation to build database queries (e.g., "SELECT * FROM users WHERE email = '" + input + "'"). The new lead architect demands all developers switch to parameterized queries immediately.

Justify the architect's decision to mandate parameterized queries to protect the application's data layer.

13. DISCUSS Error Handling [4 Marks]
Scenario: A financial trading API currently crashes and displays the raw Python stack trace, including directory paths and database structure names, when a user inputs an invalid stock ticker symbol.

Discuss the security implications of verbose error handling and how defensive error handling should be architected.
